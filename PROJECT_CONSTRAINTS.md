# 项目约束 - 必须遵守！

## 移动端优先 (Mobile-First)

**🔴 这是 ONLY for MOBILE 的项目！**

### 关键要求
- 所有页面都必须在移动端视口下设计和测试
- 默认视口：390x844 (iPhone 12/13)
- 不使用桌面端布局
- TabBar 固定在底部，符合移动端导航模式
- 触摸友好的交互设计（最小 44x44px 点击区域）
- 移动端手势支持（滑动、长按等）

### 测试规范
每次 E2E 测试必须：
1. 使用移动端 User-Agent
2. 设置 viewport 宽度 390px (或 375/360/384)
3. 验证触摸交互正常
4. 检查横向滚动（应无溢出）
5. 验证 TabBar 固定在底部且可见

### 技术决策影响
- 布局：单列为主，避免复杂网格
- 导航：底部 TabBar，非顶部菜单
- 表单：增大输入框，避免密集排列
- 字体：不小于 14px (正文)，行高至少 1.5
- 性能：减少 HTTP 请求，压缩图片

---

**⚠️ 警告：不要为桌面端设计！所有设计决策都基于移动端使用场景。**
